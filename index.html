Project 01 · Power BI · Dashboard

Celtics Ticketing
Intelligence Dashboard

A multi-page analytics dashboard built with synthetic sports data

↗ View on GitHub

This project is a comprehensive multi-page Power BI analytics dashboard built to simulate real-world sports business intelligence. Using AI-generated synthetic data across ticketing sales, social media engagement, and fan survey responses, the dashboard surfaces actionable insights about fan behavior and its relationship to ticket revenue.

The data was modeled in MySQL using relational joins across multiple tables, and DAX measures were written to calculate KPIs like fan lifetime value, retention rates, and sentiment scores. The final dashboard presents findings across dedicated report pages for Ticketing Performance, Fan Sentiment, and Social Engagement.

Key Highlights

  • Built a multi-page Power BI report with dedicated pages for ticketing, fan sentiment, and social media performance
  • Modeled relational data in MySQL using SQL joins to connect ticketing, survey, and social datasets
  • Wrote DAX measures to calculate custom KPIs including fan retention rate and ticket revenue per segment
  • Used AI-generated synthetic data to simulate realistic fan and sales patterns across a full NBA season
  • Designed interactive slicers to filter insights by game type, fan segment, and time period

Project 02 · Python · Predictive Model

NBA Points Per
Game Predictor

A live data-driven model for predicting NBA player scoring output

↗ View on GitHub

This project uses Python to build a predictive model for NBA player Points Per Game (PPG). Rather than relying on static datasets, it scrapes live data directly from Basketball-Reference, pulling current season stats for all active players to ensure the predictions reflect the most up-to-date performance.

The model factors in both offensive metrics (PPG, usage rate, shooting efficiency) and defensive context (opponent Defensive Rating, DRtg) as well as home/away game splits — since players' performance often varies significantly by venue. This results in more nuanced and accurate predictions than a simple average-based approach.

Key Highlights

  • Scraped live player stats from Basketball-Reference using Python, keeping predictions current with real season data
  • Incorporated opponent Defensive Rating (DRtg) as a key feature to account for matchup difficulty
  • Built separate models for home and away games to capture venue-based performance differences
  • Engineered features from raw stats including shooting efficiency and usage rate to improve model accuracy
  • Designed the pipeline to be rerun easily as new game data becomes available throughout the season
